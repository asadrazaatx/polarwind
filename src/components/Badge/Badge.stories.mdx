## Usage

### Default badge

Use to give a non-critical status update on a piece of information or action.

<Preview>
  <Story
    name="Default"
    decorators={[withSmartKnobs({ ignoreProps: ["children"] })]}
  >
    <Badge>{text("label", "New")}</Badge>
  </Story>
</Preview>

### Informational badge

Use to call out an object or action as having an important attribute. For example, marking
an option as "Recommended".

<Preview>
  <Story name="Informational">
    <Badge status="info">Recommended</Badge>
  </Story>
</Preview>

### Success badge

Use to indicate a successful, completed, or desirable state when it's important to provide
positive reinforcement to users.

<Preview>
  <Story name="Success">
    <Badge status="success">Success</Badge>
  </Story>
</Preview>

### Warning badge

Use to indicate a precarious state that is fixable. For example, nearing plan limits.

<Preview>
  <Story name="Warning">
    <Badge status="warning">Warning</Badge>
  </Story>
</Preview>

### Error badge

Use for critical and irreversible issues that require users' attention and potential
action.

Keep in mind that seeing this badge can feel stressful for users so it should only be used
when absolutely necessary.

<Preview>
  <Story name="Error">
    <Badge status="error">Error</Badge>
  </Story>
</Preview>
